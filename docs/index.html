Bildup

The Markdown Superset for Interactive Static Pages.

Philosophy

Bildup was born from a simple observation: Markdown is for documents, but the web is for applications.

Most "static site" solutions today require a complex build chain (Node, Webpack, React, etc.) just to get a button and a flexbox layout. Bildup bridges this gap by adding just enough syntax to Markdown to allow for layout control and interactivity without sacrificing the readability of a plain text file.

  • Zero-Build by Design: Use bildup-browser.js to render .bu files directly. No NPM needed.
  • Token Efficient: Designed to be easily generated and parsed by both humans and LLMs.
  • Self-Contained: Compiles to a single HTML file with zero external dependencies (except marked if rendering in-browser).

Comparison

vs. Standard Markdown

Markdown is great for text, but terrible for layout. In Bildup, you don't have to abuse tables for multi-column layouts or inject raw HTML <div> tags.

vs. Raw HTML

HTML is powerful but verbose. Bildup's ::[attrs] syntax is a "shorthand" for common styling patterns (padding, background, flex) that would otherwise require multiple CSS classes or inline styles.

Language Specification

1. Color Regions

Define a three-slot palette (primary, secondary, accent) scoped to a region.

^C(crimson, obsidian, amber)
This text is crimson
^C

Demo: This text is crimson, this is obsidian, and this is amber.

2. Style Regions

Wrap content in a layout container (flex or grid).

^S(flex, row, gap=4)
::[pad=2 bg=surface] Left ::
::[pad=2 bg=surface] Right ::
^S

Right Column

3. Blocks

Styled containers with padding, background, and text colors.

::[pad=4 bg=electric fg=midnight]
# High Contrast Block
::

High Contrast Block

This block uses the electric accent as a background.

4. Tables

CSV-based tables. No more pipe syntax.

::table[striped border]
Name, Role, Level
Alice, Engineer, Lead
Bob, Designer, Senior
::
NameRoleLevel
AliceEngineerLead
BobDesignerSenior

5. Interactive Components & Logic

Inputs, buttons, and calculations.

::input#val1[number value=10]/
::input#val2[number value=5]/
::button[label="Add" action=calc targets="result"]/
::output#result[label="Sum:" fn=addFn]/

::fn#addFn
  val1 + val2
::fn

Reuse components and themes.
